# ReviewPilot 接口文档

## 1. 文档说明

本文档描述 ReviewPilot MVP 阶段的本地后端接口。当前服务运行在本机，只面向前端页面调用。

基础约定：

- Base URL：`http://127.0.0.1:8770`
- 数据格式：`application/json`
- 当前不需要 GitHub Token，只支持公开 GitHub PR
- 千问 API Key 通过环境变量 `DASHSCOPE_API_KEY` 或 `QWEN_API_KEY` 读取

## 2. 健康检查

### `GET /api/health`

用于确认本地服务是否启动。

响应：

```json
{
  "ok": true
}
```

## 3. 创建 Review 分析

### `POST /api/review`

用户输入公开 GitHub PR 链接，或粘贴 unified diff。二者至少提供一个。

请求体：

```json
{
  "prUrl": "https://github.com/owner/repo/pull/1",
  "diff": ""
}
```

或：

```json
{
  "prUrl": "",
  "diff": "diff --git a/src/auth.ts b/src/auth.ts\n..."
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| prUrl | string | 否 | 公开 GitHub PR 链接 |
| diff | string | 否 | unified diff 文本 |

成功响应：

```json
{
  "pr": {
    "title": "fix login logic",
    "url": "https://github.com/owner/repo/pull/1"
  },
  "summary": "本次 PR 主要修改登录逻辑，并引入了用户身份读取方式的变化。",
  "riskLevel": "medium",
  "files": [
    {
      "path": "src/auth.ts",
      "additions": 6,
      "deletions": 0,
      "hunks": 1,
      "category": "auth"
    }
  ],
  "findings": [
    {
      "file": "src/auth.ts",
      "line": 12,
      "severity": "medium",
      "type": "security",
      "evidence": "const userId = req.body.userId;",
      "message": "新增代码直接从请求体读取用户 ID，存在越权风险，需要确认身份来源是否可信。",
      "suggestion": "建议从 session 或 token 中读取用户身份，而不是信任客户端传入的 userId。",
      "confidence": 0.82
    }
  ],
  "testSuggestions": [
    "补充未登录、伪造 userId、正常登录三类测试。"
  ],
  "evidenceCount": 8,
  "rawEvidencePreview": []
}
```

失败响应：

```json
{
  "error": "Provide a public GitHub PR URL or paste a unified diff."
}
```

常见错误：

| 错误 | 原因 | 处理方式 |
| --- | --- | --- |
| Invalid GitHub PR URL | PR 链接格式不正确 | 检查链接是否为 GitHub PR |
| Public PR not found or repository is private | PR 不存在或仓库非公开 | 改用公开 PR 或粘贴 diff |
| GitHub rate limit reached | GitHub 公开接口频率限制 | 稍后重试或粘贴 diff |
| Missing DASHSCOPE_API_KEY | 未配置千问 API Key | 设置环境变量后重启服务 |
| Qwen returned non-JSON review content | 模型返回格式不符合要求 | 后续可增加重试和修复策略 |

## 4. 后端处理流程

```text
读取请求
-> 如果提供 prUrl，则获取公开 PR diff
-> 解析 diff
-> 构建 changedFiles 和 evidence
-> 调用千问
-> 校验 findings
-> 返回 Review 报告
```

## 5. 评审结果字段

| 字段 | 说明 |
| --- | --- |
| summary | PR 变更总结 |
| riskLevel | 整体风险等级：low、medium、high |
| files | 变更文件摘要 |
| findings | 风险代码和 Review 建议 |
| testSuggestions | 测试建议 |
| evidenceCount | 传给模型的证据数量 |
| rawEvidencePreview | 证据预览，便于调试 |

## 6. 后续接口规划

- `POST /api/review/github-comment-preview`：生成模拟 GitHub 评论。
- `POST /api/review/chunked`：大型 PR 分块分析。
- `GET /api/rules`：读取仓库评审规则。
- `PATCH /api/rules`：更新评审关注点和忽略文件。
