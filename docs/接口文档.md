# ReviewPilot 接口文档

## 1. 基础约定

- Base URL：`http://127.0.0.1:8770`
- 数据格式：`application/json`
- 认证方式：登录成功后，前端在请求头中携带 `Authorization: Bearer <token>`
- GitHub：当前只支持公开仓库 PR，不需要 GitHub Token
- 千问：API Key 由用户注册或设置时提交，不需要在命令行输入

## 2. 健康检查

### `GET /api/health`

响应：

```json
{
  "ok": true
}
```

## 3. 注册

### `POST /api/register`

请求体：

```json
{
  "username": "yang070",
  "password": "secret123",
  "apiKey": "sk-...",
  "defaultModel": "qwen-plus"
}
```

响应：

```json
{
  "token": "session_token",
  "user": {
    "username": "yang070",
    "defaultModel": "qwen-plus",
    "hasApiKey": true
  }
}
```

说明：

- 账号只允许字母、数字、下划线，长度 3 到 24 位。
- 密码至少 6 位。
- 注册时必须填写千问 API Key。
- 密码不会明文保存。
- API Key 会加密保存在本地用户配置中。

## 4. 登录

### `POST /api/login`

请求体：

```json
{
  "username": "yang070",
  "password": "secret123"
}
```

响应格式同注册接口。

## 5. 当前用户

### `GET /api/me`

请求头：

```text
Authorization: Bearer session_token
```

响应：

```json
{
  "username": "yang070",
  "defaultModel": "qwen-plus",
  "hasApiKey": true
}
```

## 6. 修改设置

### `PATCH /api/settings`

请求头：

```text
Authorization: Bearer session_token
```

请求体：

```json
{
  "apiKey": "新的千问 API Key，留空则不修改",
  "defaultModel": "qwen-long"
}
```

响应：

```json
{
  "user": {
    "username": "yang070",
    "defaultModel": "qwen-long",
    "hasApiKey": true
  }
}
```

## 7. 模型列表

### `GET /api/models`

响应：

```json
{
  "models": ["qwen-plus", "qwen-plus-2025-07-28", "qwen-long", "qwen-max"]
}
```

前端还支持选择“自定义”，用户可以手动填写模型名称。

## 8. 创建 Review 分析

### `POST /api/review`

请求头：

```text
Authorization: Bearer session_token
```

请求体：

```json
{
  "prUrl": "https://github.com/owner/repo/pull/1",
  "diff": "",
  "model": "qwen-plus"
}
```

或：

```json
{
  "prUrl": "",
  "diff": "diff --git a/src/auth.ts b/src/auth.ts\n...",
  "model": "qwen-long"
}
```

成功响应：

```json
{
  "pr": {
    "title": "fix login logic",
    "url": "https://github.com/owner/repo/pull/1"
  },
  "summary": "本次 PR 主要修改登录逻辑，并引入了用户身份读取方式的变化。",
  "riskLevel": "medium",
  "model": "qwen-plus",
  "files": [
    {
      "path": "src/auth.ts",
      "additions": 6,
      "deletions": 0,
      "hunks": 1,
      "category": "auth"
    }
  ],
  "findings": [
    {
      "file": "src/auth.ts",
      "line": 12,
      "severity": "medium",
      "type": "security",
      "evidence": "const userId = req.body.userId;",
      "message": "新增代码直接从请求体读取用户 ID，存在越权风险，需要确认身份来源是否可信。",
      "suggestion": "建议从 session 或 token 中读取用户身份，而不是信任客户端传入的 userId。",
      "confidence": 0.82
    }
  ],
  "testSuggestions": [
    "补充未登录、伪造 userId、正常登录三类测试。"
  ],
  "evidenceCount": 8,
  "rawEvidencePreview": []
}
```

## 9. 常见错误

| 错误 | 原因 | 处理方式 |
| --- | --- | --- |
| 请先登录 | 未携带会话 token | 登录后重试 |
| 账号或密码错误 | 登录信息不正确 | 检查账号和密码 |
| 当前账号未配置千问 API Key | 用户没有可用 API Key | 在设置中填写 API Key |
| GitHub PR 链接格式不正确 | PR 链接格式不正确 | 检查链接是否为 GitHub PR |
| 未找到公开 PR，或该仓库不是公开仓库 | PR 不存在或仓库非公开 | 改用公开 PR 或粘贴 diff |
| GitHub 访问频率受限，请稍后重试或手动粘贴 diff | GitHub 公开接口频率限制 | 稍后重试或粘贴 diff |
| 千问返回了非 JSON 格式的评审内容 | 模型返回格式不符合要求 | 后续可增加重试和修复策略 |

## 10. 后端处理流程

```text
登录用户
-> 读取账号保存的千问 API Key
-> 获取 PR diff 或读取粘贴 diff
-> 解析变更文件
-> 构建证据链
-> 使用本次选择的模型调用千问
-> 校验 findings
-> 返回中文 Review 报告
```
