# ReviewPilot 接口文档

## 1. 基础约定

- Base URL：`http://127.0.0.1:8770`
- 数据格式：`application/json`
- 认证方式：登录成功后，前端在请求头中携带 `Authorization: Bearer <token>`
- GitHub：当前只支持公开仓库 PR，不需要 GitHub Token
- 千问：API Key 由用户注册或设置时提交，不需要在命令行输入

## 2. 注册与登录

### `POST /api/register`

```json
{
  "username": "yang070",
  "password": "secret123",
  "apiKey": "sk-...",
  "defaultModel": "qwen-plus"
}
```

### `POST /api/login`

```json
{
  "username": "yang070",
  "password": "secret123"
}
```

成功响应：

```json
{
  "token": "session_token",
  "user": {
    "username": "yang070",
    "defaultModel": "qwen-plus",
    "hasApiKey": true
  }
}
```

## 3. 设置

### `PATCH /api/settings`

```json
{
  "apiKey": "新的千问 API Key，留空则不修改",
  "defaultModel": "qwen-long"
}
```

## 4. 创建 Review 分析

### `POST /api/review`

请求头：

```text
Authorization: Bearer session_token
```

请求体：

```json
{
  "prUrl": "https://github.com/owner/repo/pull/1",
  "diff": "",
  "model": "qwen-plus"
}
```

## 6. 上下文策略

后端不会简单把 diff 从头截断，而是根据文件优先级选择重点上下文：

- 优先分析 `src/`、`app/`、`server/`、`backend/`、`frontend/` 下的源代码。
- 优先分析 `router`、`store`、`context`、`provider`、`api`、`service`、`auth`、`permission` 等路径。
- 降低 `package-lock.json`、`yarn.lock`、`pnpm-lock.yaml` 的权重。
- 降低 `dist/`、`build/`、静态图片资源的权重。
- 如果证据数量超过上限，会设置 `context_coverage.context_truncated = true`，并在 `limitations` 中提示人工复核。

## 7. 常见错误

| 错误 | 原因 | 处理方式 |
| --- | --- | --- |
| 请先登录 | 未携带会话 token | 登录后重试 |
| 账号或密码错误 | 登录信息不正确 | 检查账号和密码 |
| 当前账号未配置千问 API Key | 用户没有可用 API Key | 在设置中填写 API Key |
| GitHub PR 链接格式不正确 | PR 链接格式不正确 | 检查链接是否为 GitHub PR |
| 未找到公开 PR，或该仓库不是公开仓库 | PR 不存在或仓库非公开 | 改用公开 PR 或粘贴 diff |
| 千问返回了非 JSON 格式的评审内容 | 模型返回格式不符合要求 | 后续可增加重试和修复策略 |
